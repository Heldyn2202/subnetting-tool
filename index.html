📘 GUÍA COMPLETA DE SUBNETTING VLSM - PASO A PASO
🔹 PASO 1: Calcular los bits necesarios para hosts (n)
Fórmula: 2ⁿ - 2 ≥ Hosts requeridos
📌 Ejemplo: Para 50 hosts → probamos: 2⁵=32 (32-2=30 ❌ no alcanza), 2⁶=64 (64-2=62 ✅ alcanza)
✅ Resultado: n = 6 bits para hosts
💡 ¿Por qué restamos 2? Porque la primera IP es Network y la última es Broadcast, no asignables a dispositivos.
🔹 PASO 2: Calcular el CIDR (Prefijo de red)
Fórmula: CIDR = 32 - n
📌 Ejemplo: n=6 bits para hosts → CIDR = 32 - 6 = /26
💡 El CIDR indica cuántos bits de la dirección son para la red (los primeros 26 bits). Los restantes 6 bits son para hosts.
🔹 PASO 3: Calcular la Máscara de Red
Fórmula: Poner en "1" los primeros CIDR bits, luego convertir a decimal.
📌 Ejemplo: CIDR /26 → 26 bits en 1 = 11111111.11111111.11111111.11000000
✅ Máscara: 255.255.255.192 (los últimos 6 bits son 0 → 2⁶=64 hosts por bloque)
💡 Cálculo del último octeto: 192 = 256 - 64 (donde 64 = 2ⁿ = 2⁶)
🔹 PASO 4: Calcular el Tamaño del bloque (SALTO)
Fórmula: Salto = 2ⁿ (cantidad total de direcciones del bloque)
📌 Ejemplo: n=6 → Salto = 2⁶ = 64 direcciones
⚠️ Caso especial: Si el salto da 256, en realidad es 0 en ese octeto y suma 1 al octeto anterior.
💡 El salto se aplica en el OCTETO donde está el último bit de red (según CIDR).
🔹 PASO 5: Determinar el OCTETO de salto
- Si CIDR está entre 25 y 32 → salto en 4to octeto (último)
- Si CIDR está entre 17 y 24 → salto en 3er octeto
- Si CIDR está entre 9 y 16 → salto en 2do octeto
📌 Ejemplo: /26 → está entre 25-32 → salto en 4to octeto = 64
🔹 PASO 6: Calcular el Network ID
Fórmula: Network = Red anterior + Salto
📌 Ejemplo: Red anterior 172.20.0.0, salto 64 → Network = 172.20.0.0
📌 Siguiente red: 172.20.0.0 + 64 = 172.20.0.64
💡 La primera red siempre es la IP base que nos dan.
🔹 PASO 7: Calcular el Broadcast
Fórmula: Broadcast = Network + Salto - 1
📌 Ejemplo: Network 172.20.0.0, salto 64 → Broadcast = 172.20.0.0 + 63 = 172.20.0.63
💡 El Broadcast es la última IP del bloque, no asignable a dispositivos.
🔹 PASO 8: Calcular Primera IP Útil
Fórmula: Primera IP Útil = Network + 1
📌 Ejemplo: Network 172.20.0.0 → Primera útil = 172.20.0.1
🔹 PASO 9: Calcular Última IP Útil
Fórmula: Última IP Útil = Broadcast - 1
📌 Ejemplo: Broadcast 172.20.0.63 → Última útil = 172.20.0.62
🔹 REGLA DE ORO VLSM
⚠️ Siempre ordenar las subredes de MAYOR a MENOR cantidad de hosts antes de asignar direcciones.
💡 Esto evita fragmentación y asegura que las redes más grandes tengan espacio suficiente.
⚡ RESUMEN DE FÓRMULAS CLAVE:
• n = bits para hosts → 2ⁿ - 2 ≥ hosts_necesarios
• CIDR = 32 - n
• Salto = 2ⁿ
• Máscara = 256 - Salto (en el octeto donde ocurre el salto)
• Broadcast = Network + Salto - 1
• Rango útil = [Network+1 , Broadcast-1]
